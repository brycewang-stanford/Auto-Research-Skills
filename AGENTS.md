# Agent Coordination

This repository is often edited by multiple agents at once. Keep changes small,
observable, and easy to merge.

## Before editing

- Run `git status --short --branch` and inspect any dirty files before choosing
  a work area.
- Do not overwrite or revert changes you did not make. If a dirty file is
  relevant, read it first and make the smallest compatible edit.
- Prefer root-level scripts, docs, and generated catalog updates for maintenance
  work. Treat `skills/`, `systems/`, `benchmarks/`, and `lists/` as vendored
  submodules unless the task is explicitly to update a submodule pointer.

## Generated files

- `catalog/skills.json`, `catalog/skills.csv`, `catalog/CATALOG.md`, and
  `catalog/collections/*.md` are generated by `python tools/build_catalog.py`.
  Do not hand-edit them.
- After adding, removing, or updating submodules under `skills/`, regenerate the
  catalog and update README headline counts when `scripts/check-repo.py` asks
  for it.
- See [`docs/TOOLING.md`](docs/TOOLING.md) for the full generated-vs-source file
  map, the data pipeline, and the `make`/CI check matrix.
- See [`docs/landscape-2026.md`](docs/landscape-2026.md) for the field map that
  explains the `skills/`/`systems/`/`benchmarks/`/`lists/` split and where
  vendored projects sit in the wider ecosystem. New-vendoring decisions are
  logged under `docs/` (latest: [`docs/vendoring-2026-06-23.md`](docs/vendoring-2026-06-23.md)).

## Validation

Run these before handing off repo-level maintenance changes:

```bash
make check
```

Equivalent individual commands:

```bash
python scripts/check-repo.py
python -m unittest discover -s tests
python tools/build_catalog.py --check
./scripts/count-skills.sh
```

Use `make safety-scan SAFETY_ROOTS=<path>` for candidate skill collections.
Findings are manual-review prompts, not automatic rejection.

## Handoff

- Summarize which files you changed and which checks passed.
- Mention known warnings separately, especially the current
  `skills/claude-scholar` nested-gitlink warning.
